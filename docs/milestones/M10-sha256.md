# Milestone M10: SHA-256 Integrity Verification

## Summary
Created SHA-256 hashing utilities using the Web Crypto API (`crypto.subtle.digest`) and integrated them into the `useFileTransfer` hook. The sender now computes the file hash before chunking and includes it in FILE_META; the receiver verifies the hash from reassembled chunks during VERIFY_REQUEST processing.

New total: **97 tests passing** (50 client + 33 server + 14 shared)

## Files Added
- `packages/client/src/services/sha256.ts` — SHA-256 hash utilities (compute, convert, compare)
- `packages/client/src/__tests__/sha256.test.ts` — 14 unit tests

## Modified
- `packages/client/src/hooks/useFileTransfer.ts` — real hash computation and comparison

## Design Decisions
- **Pre-chunking hash**: sender reads the full file into memory via `file.arrayBuffer()` and hashes it before the chunking loop. This is acceptable for MVP; large-file streaming hash can be added later via web-streams-polyfill.
- **Receiver-side reassembly**: VERIFY_REQUEST handler collects all chunks in sequence order, concatenates them, and computes a single hash. This is O(n) in total data size.
- **Store feedback**: sender stores hash as hex string via `setSha256Hash`; receiver stores its computed hash on mismatch via `setReceiverHash` for UI comparison.
- **BufferSource typing**: `computeSHA256` accepts `BufferSource` (aligns with Web Crypto API signature).
